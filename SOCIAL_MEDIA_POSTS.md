### Option 2: First Principles Approach
```
Here's a first-principles question: why do we still copy/paste between AI coding tools?

When Claude Code writes code, I want a second AI opinion. But switching to Codex means:
- Copy all changed files
- Switch terminals
- Paste into new session
- Copy feedback back
- Repeat

That's 10 steps for something that should be 1.

So I built a Claude Code skill that makes Codex reviews native. Type `/codex`, get instant review.

Two benefits:

1. Speed - Proposer-checker workflow in one terminal
2. Quality - Independent AI review catches what the implementer misses

Open sourced it here: github.com/adampaulwalker/codex-claude-skill

Works with any git repo. Installation is one curl command.

The best workflows eliminate unnecessary friction.
```

### Option 3: Personal Story + Lesson
```
I almost shipped buggy test code yesterday.

Claude Code wrote 2,100+ lines of tests. They looked good. I was ready to deploy.
```
